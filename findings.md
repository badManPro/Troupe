# Findings

- 现有聊天页只有“模型正在做什么”的流式状态，没有回答“这轮头脑风暴现在能不能停”。因此需要新增一层业务语义，而不是继续堆在 `chatStatus` 上。
- `brainstorm` 阶段的收口标准在产品经理角色提示里已经隐含存在：目标用户、痛点、价值主张、差异化、MVP 边界，以及当信息足够清晰时推进到需求定义。把这些点转成前端可见的检查项是最直接的实现路径。
- 这个需求不需要改数据库。只用当前会话消息做轻量分析，就能在 UI 上实时给出“当前进度 / 可停线 / 理想线 / 还差什么”的提示。
- 对用户而言，“百分比”本身不够，必须同时给“预计还需几轮”或“现在可以停”的结论，否则还是会继续盲聊。
- 如果聊天中已经出现 `PRD`、`低保真页面说明`、`业务流程` 这类结构化内容，应显著提升收敛度，因为这说明想法已经开始沉淀成可交付产物。
