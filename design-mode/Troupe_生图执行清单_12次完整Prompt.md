# Troupe 生图执行清单：12 次完整 Prompt（可直接复制）

这份文档用于将 Troupe UI 重构需求拆成多次生图任务。  
推荐方式是：**第 1 次开新会话，后续第 2～12 次都在同一个会话里连续生成**。

---

## 使用规则

### 会话规则

- **第 1 次：新会话**
- **第 2～12 次：都在同一个会话里继续聊**

### 执行规则

- **上一张满意了，再做下一张**
- 如果某一张不满意：
  - 先在当前会话里修到满意
  - 再继续下一张
- 每次都强调：
  - 保持上一张的风格语言
  - 不要把别的页面混进来
  - 只做当前这一张

### 统一视觉方向

- **风格**：Apple HIG 克制 + Linear 风 + 编辑器感 + 轻 Bento 分区
- **配色**：低饱和中性底色 + 冷靛蓝主强调 + 六阶段主题色
- **字体**：Inter / Geist Sans + Geist Mono；中文高可读
- **密度**：中等偏高，但有呼吸感
- **圆角**：大卡片 18–24px，中小控件 10–14px，tab / badge 用胶囊
- **插画**：抽象几何 + 线性矢量，小而克制
- **动效表达**：用注释标出来，不是真的动画
- **文字要求**：尽量短文本，少用大段长文，避免把整个 PRD 全写进 UI

## 第 1 次：Design System / 风格锚点板

### 会话方式

**新会话**

### 直接复制这段

```text
请为 Troupe 设计一张高保真桌面应用 Design System / 风格锚点板，用于后续所有页面的统一视觉基准。

项目背景：
Troupe 是一个面向独立开发者 / 产品人的本地桌面级 Web 应用。用户输入一个模糊的产品想法后，由一支 AI 产品团队（7 个角色）在 6 个阶段中逐步陪伴用户把想法推进成可交付的产品方案。它既像聊天工具，也像工作台，也像文档编辑器，不是纯 Chatbot，也不是纯 IDE，而是介于两者之间的专业产品工作环境。

视觉方向：
- Apple HIG 克制 + Linear 风 + 编辑器感 + 轻 Bento 分区
- 低饱和中性底色 + 冷靛蓝主强调 + 六阶段柔和主题色
- 字体气质：Inter / Geist Sans + Geist Mono，中文高可读
- 信息密度：中等偏高，但通过留白、边框、阴影和分组保持呼吸感
- 大卡片圆角 18–24px，中小控件圆角 10–14px，tab / badge 使用胶囊圆角
- 插画与空状态图使用抽象几何 + 线性矢量，不用 3D 粘土，不用 emoji 主导
- 风格要专业、安静、有创造力，不要做成普通 SaaS Dashboard 模板

请生成一张 1440×900 的高保真设计系统板，内容包括：
1. Light / Dark 两套主题色板
2. 六阶段颜色系统：
   - 头脑风暴
   - 需求定义
   - 设计
   - 架构
   - 开发规划
   - 交付准备
3. 七个 AI 角色头像系统（小圆头像）：
   - PM
   - UI 设计师
   - 架构师
   - 前端
   - 后端
   - QA
   - 协调员
   每个角色都要有独立图标语言和颜色组合
4. 组件系统：
   - 按钮（主按钮 / 次按钮 / 危险按钮 / loading）
   - 输入框 / textarea / 搜索框
   - Tab / 胶囊标签 / Badge
   - 卡片 / Tooltip / Modal
   - Progress bar / Skeleton / Scrollbar
   - Focus ring / 选中态 / Hover 态
5. 文档状态 Badge：
   - 已确认（绿）
   - 沿用旧稿（暖黄）
   - 待产出（中性灰）
6. 桌面应用窗口头：带 macOS / 桌面应用感的顶部拖拽区和氛围光
7. 用小注释标出这些微交互表现方式：
   - hover reveal
   - selected highlight
   - expanded / collapsed
   - streaming
   - loading
   - editable message
   - focus ring

要求：
- 这是后续页面的唯一风格锚点，请做得完整、统一、可复用
- 中文文案尽量短，避免大段正文
- 高保真、产品级、可直接作为后续页面设计基准
```

## 实际操作节奏

最稳妥流程：

1. 先做 **第 1 次**
2. 如果第 1 次不满意，就继续在当前会话里修第 1 次
3. 第 1 次满意后，再按顺序做第 2、3、4……
4. 中途如果某一张跑偏：
   - 不要硬往后做
   - 先在当前会话修正这一张
   - 修满意再继续

## 通用微调模板

如果某一张生成后想微调，可以直接在同一会话追加：

```text
保持整体布局不变，只做以下调整：
1. …
2. …
3. …

其余内容保持不变。
```

适合用于：

- 调整留白
- 调整卡片密度
- 调整深色层次
- 调整阶段色
- 调整某个弹窗的信息层级
- 修正文案可读性
- 修正组件一致性
